# Диагностика проблемы с иконками на сервере

## Шаги для проверки:

### 1. Проверьте структуру на сервере
```bash
# На сервере выполните:
cd ~/app
ls -la

# Должно быть:
# index.html
# assets/
# images/
```

### 2. Проверьте где находится index.html
```bash
# На сервере:
find ~/app -name "index.html" -type f
```

### 3. Проверьте доступность иконок напрямую
Откройте в браузере:
- `https://ваш-домен.com/images/cart.svg`
- `https://ваш-домен.com/images/trash.svg`

Если открываются - проблема в коде/кэше
Если 404 - проблема в структуре папок на сервере
Если скачиваются вместо отображения - проблема с MIME-типом

### 4. Проверьте консоль браузера
Откройте Developer Tools (F12) → Console и Network
Посмотрите ошибки загрузки ресурсов

### 5. Очистите кэш браузера
- Ctrl+Shift+R (hard reload)
- Или откройте в режиме инкогнито

### 6. Проверьте конфигурацию веб-сервера

#### Для Nginx:
```nginx
# В конфиге должно быть:
location / {
    root /root/app;  # или ~/app
    try_files $uri $uri/ /index.html;
}

# И для SVG:
types {
    image/svg+xml svg svgz;
}
```

#### Для Apache:
```apache
# .htaccess должен содержать:
AddType image/svg+xml .svg .svgz
```

## Решение:

Если проблема в структуре папок, убедитесь что на сервере:
```
/root/app/
  ├── index.html
  ├── assets/
  │   ├── index-*.js
  │   └── index-*.css
  └── images/
      ├── cart.svg
      ├── trash.svg
      └── ... остальные файлы
```

Веб-сервер должен отдавать файлы из папки `/root/app/` (или где у вас находится index.html).
